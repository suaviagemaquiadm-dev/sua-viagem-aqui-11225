# Firebase Studio

Este é um projeto Next.js no Firebase Studio.

---

## Como Publicar o Site (Método Manual Definitivo)

O envio do código pelo terminal (`git push`) não está funcionando devido a um problema de autenticação no ambiente. Para resolver isso e publicar seu site, siga estes passos para enviar os arquivos manualmente.

### Passo 1: Baixar o Projeto para o seu Computador

Primeiro, precisamos baixar todos os arquivos do projeto para a sua máquina local.

1.  No painel esquerdo do Firebase Studio, clique com o botão direito na pasta principal do seu projeto (geralmente chamada `studio` ou `workspace`).
2.  Selecione a opção **"Download"**.
3.  Um arquivo `.zip` será baixado para o seu computador.
4.  Encontre o arquivo baixado e **descompacte-o**. Você terá uma pasta com todo o código do projeto.

### Passo 2: Fazer o Upload dos Arquivos para o GitHub

Agora, vamos enviar esses arquivos para o seu repositório no site do GitHub.

1.  Acesse a página do seu repositório no GitHub:
    **[https://github.com/suaviagemaquiadm-dev/sua-viagem-aqui-11225](https://github.com/suaviagemaquiadm-dev/sua-viagem-aqui-11225)**

2.  Na página do repositório, clique no botão **"Add file"** e depois selecione **"Upload files"**.

3.  Você verá uma área para arrastar arquivos. **Arraste a pasta descompactada** (com todo o seu projeto) para dentro dessa área no navegador. O GitHub irá processar e listar todos os arquivos.

4.  Aguarde até que o upload de todos os arquivos seja concluído.

5.  No final da página, você verá um campo chamado **"Commit changes"**. Escreva uma mensagem curta, como `Versao final via upload manual`.

6.  Clique no botão verde **"Commit changes"**.

### Passo 3: Aguardar a Publicação

É isso! Ao fazer o "Commit changes", o GitHub irá notificar o Firebase, que começará a publicar a nova versão do seu site.

Aguarde de 3 a 5 minutos. Após esse tempo, acesse a URL da sua aplicação, e o site estará no ar e atualizado.
